# KiroClaim Commerce Implementation Plan

**Goal:** Deliver a usable guest card storefront and administrator commerce console.

**Architecture:** Add focused commerce models and handlers to the existing Gin/GORM monolith. Keep payment, fulfillment, proof storage, and notification boundaries separate while sharing a transactional order state machine.

**Tech Stack:** Go, Gin, GORM, SQLite/MySQL, vanilla HTML/CSS/JavaScript.

## Tasks

1. Add commerce models, constants, automatic migration, and state-transition tests.
2. Add catalog and channel administration with encrypted secrets.
3. Add public product, order creation, secure query, proof upload, and callback endpoints.
4. Add transactional generated and pre-generated fulfillment, expiration, review, and recovery actions.
5. Add local/S3-compatible proof storage and SMTP configuration/notification services.
6. Add `/shop` storefront and `/shop-admin` administrator console.
7. Run focused tests, full tests, build, server smoke tests, and browser validation.
